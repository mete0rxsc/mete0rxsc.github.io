GitAutumn

和GitHub用法其实差不多的，不过还是写一下推送上传的一些教程吧。

创建仓库 ​

在 GitAutumn上创建一个新的仓库，信息什么的自己弄。

配置密匙 ​

git安装 ​

Debian: sudo apt install git Fedora: sudo dnf install git 可通过git --version 命令查看，正常输出git版本号即安装成功。

配置用户名和邮箱名 ​

命令如下:

git config --global user.name "Your Name"
git config --global user.email "email@example.com"

配置完后可以通过以下命令查看:

git config user.name
git config user.email

配置SSH公钥 创建公钥文件，生成的公钥文件会保存在本机的~/.ssh目录下: ssh-keygen -t rsa -b 4096 -C "your_email@example.com" 然后三步询问直接回车即可(连续三次回车)。

创建完成后进~/.ssh目录，(将公钥即id_rsa.pub的内容)复制出来:

cd ~/.ssh
cat id_rsa.pub

如果进入该目录时提示No such file or directory，自己手动创建一个即可:mkdir ~/.ssh

然后回到最初的SSH / GPG 密钥 填写你复制的密匙。

这一段素材教程提供来自：青稚の空间 - 越努力,越幸运

克隆仓库 ​

在本地创建一个空文件夹，将 GitAutumn仓库地址复制下来，在空文件夹中右键打开 Git Bash Here，输入git clone 仓库地址，本地就会在该文件夹下看见克隆的文件名。

添加代码 ​

将要上传的项目文件复制到克隆的文件夹下。在 Git Bash Here 中，先执行初始化操作，输入git init。然后执行git add 文件名（如果是git add .表示当前目录下的所有内容），将代码添加到暂存区。注意，add后要有一个空格。

提交代码 ​

执行git commit -m "第一次提交"，其中-m后面的文本是本次提交的注释信息。

添加远程仓库 ​

执行git remote add origin 仓库地址，将本地仓库与 GitAutumn仓库关联。

推送代码 ​

执行git push -u origin master，将代码推送到 GitAutumn仓库。

master是分支，查看分支git branch

提交演示 ​

弄完前置操作后，进入克隆下来的文件夹。

输入完你的用户名和密码，然后就会开始正常上传。

然后回到仓库刷新就会发现上传完毕了。
